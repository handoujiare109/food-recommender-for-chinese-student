# 美食推荐系统

基于 **Thompson Sampling（汤普森采样）** 的美食推荐 Web 应用。本项目是概率论、数理统计与数据科学课程的课程项目，将课堂所学的贝叶斯统计、Beta 分布、多臂老虎机等理论知识应用于实际推荐场景。

---

## 课程背景

本项目涵盖了以下课程核心知识点：

### 概率论

- **Beta 分布与 Gamma 分布**：Thompson Sampling 的核心是 Beta 分布。Beta 分布是定义在 [0, 1] 区间上的连续概率分布，由两个形状参数 α 和 β 控制，非常适合对点击率等概率参数建模。算法中通过 Marsaglia-Tsang 方法从 Gamma 分布采样，再构造 Beta 随机变量。

- **共轭先验（Conjugate Prior）**：Beta 分布是二项分布的共轭先验。这意味着如果先验是 Beta(α, β)，观测到 s 次成功（点击）和 f 次失败（未点击）后，后验分布仍然是 Beta 分布：Beta(α + s, β + f)。这一性质使得贝叶斯更新变得极为简洁。

- **大数定律与不确定性量化**：随着交互次数增加，后验分布逐渐集中到真实点击率附近，体现了大数定律的思想。分布方差的大小反映了我们对估计值的不确定程度。

### 数理统计

- **贝叶斯推断（Bayesian Inference）**：不同于频率学派的点估计，本项目采用贝叶斯方法，将对菜品点击率的认知表示为一个完整的概率分布。每次用户反馈后，利用贝叶斯定理更新后验分布。

- **先验分布的构建**：系统使用领域知识为每道菜品设定了初始权重（先验均值），并通过伪计数 N0 = 20 控制先验强度。这相当于假设在观察到任何真实数据之前，已经进行了 20 次虚拟试验。

- **后验期望估计**：每个菜品的后验期望值为 (α + clicks) / (α + β + shows)，这是对其真实点击率的最小均方误差估计。

### 数据科学

- **探索与利用（Exploration vs. Exploitation）**：多臂老虎机问题的核心矛盾。纯利用（总是推荐当前最优）可能错过更好的选项；纯探索（随机推荐）浪费了大量机会。Thompson Sampling 通过随机采样自然地平衡两者。

- **在线学习（Online Learning）**：系统在每次交互后立即更新模型，无需离线批量训练。这是一种增量学习方式，适合实时推荐场景。

- **冷启动问题（Cold Start）**：新系统没有历史数据，通过合理的先验分布可以给出初始推荐，并随着用户反馈快速收敛。

## 工作原理

系统将 27 种常见美食按照两个维度分为四个象限：

| 价格 \ 辣度 | 不辣 | 辣 |
|---|---|---|
| 25元以下 | 第一象限 | 第二象限 |
| 25元及以上 | 第三象限 | 第四象限 |

### Thompson Sampling 算法流程

对于每次推荐请求，算法执行以下步骤：

1. 根据用户选择的价格和辣度偏好，确定对应的象限
2. 对该象限内的每道菜品，计算其后验 Beta 分布参数：
   - α = α0 + clicks（先验伪成功数 + 实际感兴趣次数）
   - β = β0 + (shows - clicks)（先验伪失败数 + 实际不感兴趣次数）
3. 从每个 Beta(α, β) 分布中随机采样一个值
4. 返回采样值最高的菜品作为推荐结果

采样值最高的菜品被推荐后，用户给出"感兴趣"或"不感兴趣"的反馈，系统据此更新该菜品的 clicks 和 shows 计数，完成一次贝叶斯更新。

## 技术栈

| 技术 | 用途 |
|---|---|
| Node.js (v18+) | JavaScript 运行时 |
| Express 5.x | Web 服务框架 |
| 原生 JavaScript (ES6+) | 前端逻辑与 ML 算法实现 |
| HTML5 + CSS3 | 用户界面 |
| JSON 文件 | 数据持久化 |

整个 Thompson Sampling 引擎（Gamma 采样、Beta 采样、后验计算）均由原生 JavaScript 从零实现，没有使用任何机器学习第三方库，便于理解算法底层原理。

## 快速开始

### 环境要求

- Node.js v18 或更高版本

### 安装

```bash
git clone https://github.com/handoujiare109/food-recommender-for-chinese-student.git
cd food-recommender-for-chinese-student
npm install
```

### 运行

```bash
npm start
# 或者
node server.js
```

服务启动后访问 `http://localhost:3000`。

| 页面 | URL | 说明 |
|---|---|---|
| 主页面 | `http://localhost:3000/` | 选择价格和辣度偏好，获取美食推荐 |
| 管理后台 | `http://localhost:3000/admin` | 查看所有菜品的统计数据（仅限本机访问） |

## API 接口

### GET /api/recommend

获取美食推荐。

查询参数：

| 参数 | 类型 | 说明 |
|---|---|---|
| price | 0 或 1 | 0 = 25元以下，1 = 25元及以上 |
| spicy | 0 或 1 | 0 = 不辣，1 = 辣 |

返回示例：

```json
{
  "name": "兰州拉面",
  "price": 0,
  "spicy": 0,
  "score": 0.723
}
```

其中 `score` 为该菜品在当前后验 Beta 分布中的采样值，代表了本次推荐中该菜品的"竞争力"。

### POST /api/feedback

提交推荐反馈，触发贝叶斯更新。

请求体 (JSON)：

```json
{
  "name": "兰州拉面",
  "interested": true
}
```

### GET /api/data

获取所有菜品的完整统计数据，包括先验权重、展示次数、点击次数、实际点击率与后验期望得分。

## 功能特性

- **Thompson Sampling**：概率匹配（probability matching）策略，每次按最优概率采样，自然平衡探索与利用
- **贝叶斯共轭先验**：利用 Beta-Binomial 共轭关系，实现简洁高效的后验更新
- **实时在线学习**：每次用户反馈后立即更新模型，无需批量重训练
- **冷启动处理**：通过领域知识先验，系统在没有任何数据时也能给出合理推荐
- **持久化存储**：数据以 JSON 格式存储，支持防抖写入，确保数据安全
- **管理后台**：实时展示所有菜品的先验权重、展示/点击次数、实际 CTR 和后验期望分数
- **零外部 ML 依赖**：所有统计算法从零手写，适合教学演示
